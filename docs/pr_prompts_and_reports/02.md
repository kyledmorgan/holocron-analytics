Enables one-command local development: `docker compose up --build` starts SQL Server 2022 Developer Edition, creates the database, runs DDL scripts, and loads seed data. Only Docker Desktop required.

### New Files

- **`docker-compose.yml`** — Three services:
  - `sqlserver`: SQL Server 2022 with persistent volume, healthcheck on port 1433
  - `initdb`: Creates database + runs DDL scripts (dimensions → facts → bridges)
  - `seed`: Builds Python container, runs `seed_loader.py --all` after initdb completes

- **`docker/Dockerfile.seed`** — Python 3.11 slim with ODBC Driver 18 for SQL Server

- **`docker/init-db.sql`** — Idempotent database creation via sqlcmd variables

- **`docs/runbooks/docker_local_dev.md`** — Full runbook: prerequisites, quick start, connection info, troubleshooting

### Modified Files

- **`.env.example`** — Added `MSSQL_SA_PASSWORD`, `MSSQL_DATABASE`, `SEED_SKIP` with clear Docker vs native sections

- **`README.md`** — Added Quick Start with Docker section linking to runbook

### Usage

```bash
cp .env.example .env  # Set MSSQL_SA_PASSWORD
docker compose up --build
```

Connect via SSMS/Azure Data Studio: `localhost,1433` | `sa` | your password

Reset everything: `docker compose down -v`
