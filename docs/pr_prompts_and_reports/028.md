## PR Body

Repository lacked README files for major directories and contributor guidance. New users had no fast-path to getting started.

## Added

**Root-level guides:**
- `QUICKSTART.md` — 5-minute Docker setup path
- `CONTRIBUTING.md` — Development workflow, style guides, PR process

**Directory READMEs:**
- `scripts/README.md` — CLI utilities (db_smoketest, llm_enqueue_job, test runners)
- `tools/README.md` — Core infrastructure (db_init migration tool)
- `exercises/README.md` — Learning exercise structure (SQL, Python, scenarios)
- `prompts/README.md` — LLM prompt template catalog
- `web/README.md` — Planned web features (currently placeholder)

**Index update:**
- `docs/DOCS_INDEX.md` — Added all new docs to "Start Here" and Full Inventory sections

## Structure

Each directory README follows consistent pattern:
- Overview and status
- Directory structure diagram
- Key files/scripts with usage examples
- Environment variables and configuration
- Related documentation links
- Troubleshooting (where applicable)

All documentation cross-references existing guides (ERD, LLM module, Docker runbook) rather than duplicating content.
